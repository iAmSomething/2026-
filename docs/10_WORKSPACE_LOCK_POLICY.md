# 작업영역 락 정책

## 목적
- 멀티 스레드 동시 작업 시 충돌을 줄이고, 역할별 책임 경계를 명확히 한다.

## 기본 원칙
1. 각 역할은 본인 작업영역 파일만 수정한다.
2. 공용 잠금 경로 수정은 PM 승인 코멘트 후 진행한다.
3. 이슈 생성 시 "작업영역 락 체크리스트"를 반드시 체크한다.

## 역할별 기본 작업영역
1. UIUX
- `apps/web/**`
- `UIUX_reports/**`
- `poll_uiux_docs_v0.1/**`

2. Collector
- `src/pipeline/**`
- `Collector_reports/**`
- `data/**` (수집/추출 산출물)

3. Develop
- `app/**`
- `db/**`
- `develop_report/**`
- `scripts/qa/**`

4. QA
- `QA_reports/**`
- `tests/**` (검증 코드)
- `scripts/qa/**` (검증 자동화)

## 공용 잠금 경로 (동시수정 금지)
1. `docs/**`
2. `.github/**`
3. `scripts/pm/**`
4. `README.md`

## 공용 잠금 경로 수정 절차
1. 이슈 코멘트로 "수정 예정 파일 + 목적 + 예상 시간" 선점 선언
2. PM 승인 코멘트 확인 후 수정
3. PR/이슈 코멘트에 승인 링크와 변경 근거 첨부

## 에스컬레이션 규칙
1. 잠금 경로 무단 수정 발견 시 즉시 `status/in-review`로 전환
2. PM이 충돌 정리 후 재할당
3. 반복 위반 시 해당 스레드는 공용 파일 수정 권한 일시 제한
