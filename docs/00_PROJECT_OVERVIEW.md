# 프로젝트 개요

- 문서 버전: v0.2
- 최종 수정일: 2026-02-18
- 수정자: Codex

## 1. 프로젝트 목적
- 지방선거 관련 기사에서 여론조사 데이터를 자동 추출/저장/정규화한다.
- 검증 가능한 근거와 함께 시계열 추세를 대중에게 직관적으로 제공한다.
- 메인 대시보드, 지역 검색, 후보자 상세 조회를 한 플랫폼에서 제공한다.

## 2. 비목표
- 당선 확률 예측 모델 제공
- 정치 성향 해석/논평 자동 생성
- 기사 전문 재배포

## 3. 핵심 사용자
- 1차 타깃: 일반 대중
- 2차 사용자: 내부 운영/검수 담당자

## 4. 확정 의사결정 로그
1. 데이터 원천 전략: `공식기준+기사보강`
2. 추출 방식: `하이브리드(규칙+LLM)`
3. 정규화 정책: `원문 + min/max/mid + 결측 동시저장`
4. 검수 정책: `반자동 승인`
5. 실행 주기: `2시간 간격`
6. UI 방향: `데이터 뉴스룸형`
7. 우선 디바이스: `데스크탑 우선`
8. 기술 축: `Python 중심 + FastAPI + Supabase + Next.js`

## 5. 범위
### 포함
1. 기사 수집 및 여론조사 기사 판별
2. 수치/문항/조사메타 추출 및 정규화
3. 지역/직책/후보 단위 시계열 조회
4. 지도 Hover 기반 광역 단위 요약
5. 기초자치단체 검색 + 선거 타입별 매치업
6. 후보자 상세(약력/출마 정보) 제공

### 제외
1. 예측 모델 및 승패 확률
2. 모바일 앱 네이티브 개발
3. 사용자 생성 데이터 편집 기능

## 6. 성공 기준
1. 핵심 지표 누락 없는 데이터 적재
2. 추출 데이터의 근거(evidence) 추적 가능
3. UI의 주요 시나리오(지도/검색/상세) 완결
4. 운영 관점에서 재처리/검수/모니터링 가능
